# UI / Feature Backlog (MVP → v1)

Цель: максимально быстро оцифровать гардероб и собирать образы без открытия шкафа.

## P0 (закрыть для комфортного MVP)
- Быстрое добавление вещи: фото → минимальные поля → сохранить (и “добавить ещё”).
- Пустые состояния на ключевых экранах с понятными CTA (“Добавить вещь”, “Создать образ”, “Запланировать”).
- Просмотр/редактирование вещи (экран детали): фото, атрибуты, где используется, смена статуса, архив.
- Стабильная навигация и понятные первичные действия (плюс-меню в Today/Wardrobe/Outfits).
- Accessibility identifiers на основных контролах (подготовка к UI тестам).

## P1 (польза заметно растёт)
- Batch-скан/импорт: режим “серия фото” + быстрые правки атрибутов (swipe/inline).
- Авто-вырезание фона + кроп/масштаб (ручная корректировка).
- Улучшенные фильтры: цвет, сезон, формальность, теги, статус, “не носил N дней”.
- “Shuffle / Remix”: сгенерировать варианты образов из чистых вещей, сохранить как Outfit.
- Календарь планов: назначение образа прямо в Planner + просмотр недели.
- Поездки/packing list: собрать чемодан из образов (по дням/активностям).

## P2 (когда MVP стабилен)
- Cost-per-wear, бюджет/стоимость, “покупки vs носки”, рекомендации “что докупить”.
- Синхронизация/бэкап: iCloud, экспорт/импорт (JSON/CSV), перенос между устройствами.
- Персонализация: любимые бренды/силуэты, запреты/аллергии, дресс-код по локациям.
- Интеграции: календарь, напоминания, погода расширенная (ветер/осадки/ощущается).
- Мультигардероб: “дом/офис/спорт”, сезонное хранение.

## UX принципы
- Первое значение за 60 секунд: добавить 1–3 вещи, увидеть рекомендации/образы.
- Минимум обязательных полей: имя можно автогенерировать, остальное — позже.
- Всё важное в 1–2 тапа: добавить вещь/создать образ/запланировать/отметить “надето”.
