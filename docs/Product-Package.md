# WardrobeFlow iPhone MVP - Product Package

## 1. Product Vision
Пользователь один раз оцифровывает гардероб и ежедневно собирает уместные образы за 30-60 секунд без открытия шкафа.

## 2. Target Audience
- Занятые пользователи 20-45 лет, которым нужен быстрый выбор одежды.
- Пользователи, которые покупают одежду импульсивно и теряют контроль над тем, что уже есть.
- Пользователи, планирующие поездки и рабочую неделю.

## 3. Core Jobs-To-Be-Done
- "Когда я собираюсь утром, я хочу быстро получить подходящий образ по погоде и планам дня."
- "Когда покупаю одежду, хочу понимать, есть ли уже похожая вещь."
- "Когда еду в поездку, хочу быстро собрать капсулу и не забыть важное."

## 4. MVP Scope (v1)
- Каталог одежды с фото, тегами и статусом (чисто/стирка/ремонт).
- Быстрое добавление вещи с полуавто-классификацией (в MVP правила + ручной выбор).
- Фильтры/поиск по типу, цвету, сезону, формальности.
- Конструктор образов и сохранение любимых образов.
- Планировщик образов на неделю.
- Базовые рекомендации по погоде (через уровень температуры).
- Аналитика: частота носки, вещи без использования 30+ дней.

## 5. Out of Scope for MVP
- Полное компьютерное зрение с точным распознаванием ткани/бренда.
- Социальные функции и шеринги.
- Маркетплейс-интеграции.
- AR-примерка.

## 6. Functional Requirements
1. Пользователь может добавить/редактировать/архивировать вещь.
2. Каждая вещь имеет минимум 1 фото и атрибуты.
3. Вещи доступны в поиске и фильтрах.
4. Пользователь может собрать образ из 2+ вещей.
5. Пользователь может назначить образ на дату.
6. Система учитывает доступность вещи (не в стирке).
7. Система помечает вещь как "надета" после использования в плане.

## 7. UX Requirements
- Важный сценарий "что надеть сегодня" открывается первым.
- Добавление вещи не дольше 20 секунд в базовом режиме.
- Любая ручная правка делается за 1-2 действия.
- Все ключевые сценарии доступны одной рукой.

## 8. Non-Functional Requirements
- Offline-first для каталога и образов.
- Синхронизация через iCloud (в следующем спринте после MVP ядра).
- Вход по Face ID/Touch ID.
- Локальное хранение фото и персональных данных.

## 9. Success Metrics
- D7 retention >= 30%.
- Не менее 15 добавленных вещей у активного пользователя за первую неделю.
- Не менее 3 сохраненных образов за первую неделю.
- Median Time To Outfit <= 60 секунд.

## 10. Key Risks and Mitigation
- Сложный первичный онбординг: пакетный режим добавления + пошаговые подсказки.
- Ошибки классификации: быстрый ручной override.
- Падение вовлеченности: ежедневный виджет "образ дня" и напоминания.

## 11. Release Plan
- Alpha (локально): каталог + конструктор + планировщик.
- Beta (TestFlight): рекомендации по погоде + аналитика + стабильность.
- v1.0: polish UI, onboarding, приватность, метрики.
