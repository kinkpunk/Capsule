# User Flows

## Flow 1 - First Launch + Onboarding
1. Открытие приложения.
2. Короткое объяснение ценности (3 экрана).
3. Разрешение на фото (камера/галерея).
4. Переход к первому добавлению вещи.

## Flow 2 - Add Item (Fast)
1. Пользователь нажимает "Добавить вещь".
2. Делает фото или выбирает из галереи.
3. Приложение предлагает категорию и цвет.
4. Пользователь подтверждает/правит.
5. Сохранение вещи в каталог.

## Flow 3 - Build Outfit
1. Открытие вкладки "Образы".
2. Выбор верх/низ/обувь/аксессуары из доступных вещей.
3. Проверка совместимости (сезон/формальность/статус).
4. Сохранение образа.
5. Опционально - назначение на дату.

## Flow 4 - Plan Week
1. Открытие вкладки "План".
2. Выбор даты.
3. Выбор готового образа или сборка нового.
4. Сохранение в календарь.
5. Утреннее напоминание о запланированном образе.

## Flow 5 - Daily Recommendation
1. Пользователь открывает "Сегодня".
2. Система получает температуру (или пользователь задает вручную).
3. Приложение предлагает 3 образа.
4. Пользователь подтверждает один образ.
5. Вещи отмечаются как использованные.

## Flow 6 - Laundry and Availability
1. После носки пользователь отмечает вещи как "в стирке".
2. Эти вещи скрываются из конструктора.
3. После стирки пользователь возвращает статус "чисто".

## Flow 7 - Trip Packing (post-MVP)
1. Пользователь создает поездку с датами и погодой.
2. Приложение предлагает капсулу.
3. Пользователь подтверждает и отмечает упакованное.
